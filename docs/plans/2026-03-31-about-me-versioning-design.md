# About Me Versioning Design

**Date:** 2026-03-31

## Goal

포트폴리오의 `About me` 문구를 하단 프로젝트/이력의 결에 맞게 재정렬하되, 기존 문구는 `v1`으로 보관하고 현재 화면에는 `v2`만 노출한다.

## Context

현재 `About me`는 소통과 성장의 가치를 잘 담고 있지만, 아래 프로젝트 섹션이 보여주는 강점인 문제 구조화, 검증 기준 수립, 성능/품질 판단, AI+백엔드 구현 경험이 상대적으로 약하게 드러난다.

## Design

- `index.html`의 `About me` 섹션 내부에 `v1`, `v2` 문구를 모두 보관한다.
- `v1`은 향후 재사용을 위해 DOM 안에 남기되 화면에는 보이지 않게 처리한다.
- `v2`는 기존 `v1`의 문체와 분량을 유지하면서도, 아래 이력의 공통 축인 `판단 기준`, `검증`, `문제 구조화`, `실제 서비스 구현`을 반영한다.
- `css/style.css`에는 현재 활성 버전만 드러나도록 최소한의 스타일 클래스만 추가한다.

## Chosen Copy Direction

`개발은 단순히 기능을 구현하는 일이 아니라, 문제를 올바르게 정의하고 더 나은 답을 찾아가는 과정이라고 생각합니다. 그렇기에, 저는 감에 의존하기보다 먼저 판단의 기준을 세우고 성능과 품질을 검증하며 해결 방향을 선택하는 태도를 중요하게 생각합니다. 프로젝트를 진행하며 문제의 본질을 구조화하고, AI와 백엔드 기술을 연결해 실제로 작동하는 서비스를 만들어왔습니다. 그렇게 기준을 세우고 검증해가는 과정 속에서 더 나은 결과를 만들어내는 즐거움을 느끼며, 팀과 함께 근거 있는 선택을 쌓아가는 개발자로 성장하고자 합니다.`
