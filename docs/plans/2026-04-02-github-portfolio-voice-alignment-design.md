# GitHub Portfolio Voice Alignment Design

**Date:** 2026-04-02

## Goal

`smj1513.github.io`와 GitHub 프로필 `smj1513/README.md`를 함께 보았을 때, 같은 개발자를 소개하는 문서처럼 자연스럽게 이어지도록 톤과 첫인상 문구를 정렬한다.

## Current Findings

- 포트폴리오는 `문제 정의 → 판단 기준 수립 → 검증 → 실제 서비스 구현`의 흐름이 분명하다.
- 현재 GitHub 프로필 README는 배지와 스택 중심 구조는 깔끔하지만, 소개 문구가 `관심사 나열`에 더 가까워 포트폴리오의 문제 해결형 인상과 결이 다소 다르다.
- 포트폴리오 상단의 짧은 한 줄 소개는 아직 이전 톤인 `배움·소통·성장` 축이 강해서, 바로 아래 `About me v2` 및 프로젝트 서사와 완전히 맞물리지는 않는다.

## Chosen Direction

- 구조는 유지한다.
  - GitHub README의 배지, 통계 카드, 스택 블록, 학력/경험 블록은 그대로 둔다.
  - 포트폴리오의 전체 레이아웃과 프로젝트 섹션도 그대로 둔다.
- 문구만 정렬한다.
  - GitHub README의 인사말과 관심사 키워드를 포트폴리오의 핵심 축에 맞춰 다시 쓴다.
  - 포트폴리오 상단 한 줄 소개는 `소통하며 성장` 중심 표현에서 `문제 정의와 검증을 바탕으로 구현하는 개발자` 쪽으로 소폭 이동시킨다.

## Copy Principles

- 과장보다 태도를 보여주는 문장으로 쓴다.
- `관심이 많습니다`보다 `어떻게 일하는가`가 드러나는 표현을 우선한다.
- 포트폴리오의 핵심 키워드인 `문제 정의`, `판단 기준`, `검증`, `AI와 백엔드의 연결`, `실제 서비스 구현`을 GitHub README에도 반영한다.
- GitHub 프로필은 여전히 가볍게 읽히도록 문단 길이는 짧게 유지한다.

## Editing Scope

- Modify: `index.html`
- Update remotely: `smj1513/README.md`

## Expected Result

- GitHub 프로필을 먼저 보면 `근거 있게 판단하고 검증하며 구현하는 백엔드 개발자`라는 인상이 남는다.
- 이후 포트폴리오를 보면 같은 메시지가 프로젝트 사례로 자연스럽게 확장된다.
- 반대로 포트폴리오를 먼저 봐도 GitHub README가 더 이상 별개의 톤으로 느껴지지 않는다.
