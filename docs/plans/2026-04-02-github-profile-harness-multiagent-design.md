# GitHub Profile Harness Engineering Update Design

**Date:** 2026-04-02

## Goal

GitHub 프로필 README의 현재 소개 흐름을 유지하면서, 최근 관심사로서 `하네스 엔지니어링`과 `멀티에이전트 시스템`을 자연스럽게 드러낸다.

## Context

- 현재 README는 `문제 정의`, `검증`, `AI+백엔드`, `검색/RAG/LLM 운영` 축으로 정리되어 있다.
- 사용자는 여기에 `하네스 엔지니어링`과 `멀티에이전트 시스템`도 흥미와 내용 축으로 포함하길 원한다.
- 다만 구조를 무겁게 바꾸지 않고, 가볍고 배지 중심인 인상은 유지해야 한다.

## Chosen Direction

- 소개 문단의 마지막 문장에 `에이전트 하네스 엔지니어링`과 `멀티에이전트 시스템 설계`를 함께 녹인다.
- `Interests`는 3개 항목 수를 유지하되, 하나의 항목에 `Harness Engineering`, 다른 항목에 `Multi-Agent Systems`를 반영한다.
- 포트폴리오 상단 소개 문구는 수정하지 않는다.

## Copy Principles

- 새 키워드는 기존 문구 흐름에 덧붙이는 방식으로 넣는다.
- 단순 키워드 나열보다 `실제 서비스`, `운영 품질`, `설계`와 연결되도록 쓴다.
- README 첫머리가 과하게 빽빽해지지 않도록 문장 수와 항목 수는 유지한다.

## Expected Result

- GitHub README를 봤을 때, 검색/RAG/LLM 운영뿐 아니라 에이전트 하네스와 멀티에이전트 시스템에도 관심이 있다는 점이 자연스럽게 보인다.
- 기존의 문제 해결형 톤과 배지 중심 레이아웃은 그대로 유지된다.
