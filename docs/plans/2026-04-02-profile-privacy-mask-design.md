# Profile Privacy Mask Design

**Date:** 2026-04-02

## Goal

첫 번째 프로필 카드에 개인정보 마스킹 버튼을 추가해, `GitHub`와 `Hugging Face`를 제외한 카드 내부 정보를 한 번에 가리거나 다시 보여줄 수 있게 만든다.

## Scope

- 유지 대상:
  - `GitHub` 링크
  - `Hugging Face` 링크
- 마스킹 대상:
  - 프로필 이미지
  - 이름과 영문 이름
  - 한 줄 소개
  - `Birth`
  - `E-Mail`

## Chosen Approach

- 프로필 카드에 토글 버튼을 추가한다.
- 카드에 `is-masked` 클래스를 토글하는 방식으로 마스킹 상태를 제어한다.
- 마스킹 대상 요소에는 공통 클래스를 부여하고, CSS로 블러/숨김/플레이스홀더 스타일을 적용한다.
- `GitHub`와 `Hugging Face` 행은 마스킹 대상에서 제외한다.

## Interaction Design

- 기본 상태는 `마스킹 해제`다.
- 버튼 클릭 시 `is-masked` 클래스가 카드에 붙고, 버튼 텍스트는 `개인정보 보기`로 바뀐다.
- 다시 클릭하면 원래 상태로 돌아간다.
- 버튼은 프로필 카드 내부 상단에 배치해 시선 흐름을 방해하지 않도록 한다.

## Expected Result

- 방문자는 필요할 때만 카드 내 개인정보를 가릴 수 있다.
- 공개 링크는 계속 보이므로 포트폴리오 탐색 동선은 유지된다.
- 구현은 HTML/CSS/짧은 JS만으로 유지되어 정적 페이지 구조를 해치지 않는다.
