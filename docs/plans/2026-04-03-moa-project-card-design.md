# MOA Project Card Design

**Date:** 2026-04-03

## Goal

포트폴리오에 `MOA` 프로젝트 카드를 새로 추가하되, 카드의 배치와 설명 축은 `전체 서비스` 기준으로 잡고, 세부 역할과 기술 기여는 `MOA-Inference` 서버 내용을 기반으로 정리한다.

## Framing

- 카드 제목과 서비스 개요는 `목표 기반 자산관리 금융 서비스` 관점으로 작성한다.
- 역할 및 기여, 고민 및 구현은 `MOA-Inference` 저장소에 정리된 AI 서버 개발 내용을 중심으로 구성한다.
- 기존 프로젝트 카드들의 구조와 톤을 유지한다.

## Chosen Layout

- 위치: 기존 `Projects` 섹션의 맨 위
- 카드 제목: `MOA - 목표 기반 자산관리 금융 페이스메이커`
- 날짜: `2026.02 ~ 2026.04`
- 대표 이미지: `assets/images/MOA/Screenshot_20260401_150601_Gallery.jpg`를 기반으로 가로형 대표컷을 새로 가공해 사용
- 실제 사용자 화면: 여러 모바일 화면을 한 장의 와이드 쇼케이스 이미지로 가공해 사용
- 보조 갤러리: 목표 상세 화면, 주간 현황, AI 코치 화면 등 3~4장

## Content Direction

- 서비스 개요:
  - 기존 가계부가 기록과 통계에 머무르는 문제
  - 목표 기준으로 현재 소비 수준, 자유소비 가능 금액, 조정 우선순위를 판단하게 돕는 서비스
  - Flutter 앱 + Spring Boot 백엔드 + FastAPI AI 서버 구조
- 역할 및 기여:
  - KoELECTRA attention pooling 기반 가맹점 카테고리 분류 모델 개선
  - SQLToolkit 기반 구조를 Spring AI MCP 기반 소비 코칭 에이전트 구조로 전환
  - MCP 컨텍스트 주입, 스트리밍 응답, 대화 메모리 설계
  - 이미지 생성 relay 엔드포인트 구성
- 고민 및 구현:
  - LLM 분류의 비결정성 문제
  - SQL 생성 기준과 서비스 정책 기준 불일치 문제
  - 이미지 생성 입력 검증과 relay 분리 필요성

## Image Direction

- 대표 이미지는 진입화면 스크린샷을 그대로 쓰지 않고, 카드 메인 이미지 비율에 맞춰 배경과 함께 정리된 가로형 썸네일로 재가공한다.
- 쇼케이스 이미지는 모바일 화면 3장을 묶은 가로형 구성으로 만들어, 기존 카드의 대표 화면 영역과 시각적 결을 맞춘다.
- 세로형 원본 이미지는 보조 갤러리에서 `object-fit` 기반으로 정리한다.

## Expected Result

- 사용자는 카드 첫인상에서 `MOA`를 하나의 완성된 서비스로 인지한다.
- 동시에 카드 본문을 읽으면 내가 맡은 AI 서버/추론/분석 파트 기여가 분명하게 드러난다.
- 이미지 톤도 기존 프로젝트 카드와 어색하지 않게 이어진다.
