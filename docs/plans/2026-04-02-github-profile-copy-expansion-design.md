# GitHub Profile Copy Expansion Design

**Date:** 2026-04-02

## Goal

GitHub 프로필 README의 소개 문단과 `Interests`를 지금보다 더 풍성하게 다듬되, 현재의 가볍고 배지 중심인 구조는 그대로 유지한다.

## Current Gap

- 현재 소개 문구는 방향성은 맞지만 `태도` 중심이라, 어떤 기술 영역에서 그 태도를 발휘하는지까지는 충분히 전달되지 않는다.
- `Interests`는 포트폴리오와 축은 맞지만 표현이 다소 추상적이라, 검색/RAG/LLM 운영이나 성능 검증 같은 실제 경험과의 연결감이 약하다.
- 결과적으로 프로필을 봤을 때 `어떤 사람인가`는 보이지만 `어떤 문제를 어떤 영역에서 다루는가`는 한 단계 덜 선명하다.

## Chosen Direction

- 소개 문단은 2문장에서 3문장으로 확장한다.
- 문장 흐름은 다음 순서를 따른다.
  - 문제 정의와 검증을 중시하는 태도
  - AI와 백엔드를 연결해 실제 서비스로 구현하는 방식
  - 검색/RAG/LLM 시스템을 운영 품질과 사용자 경험까지 포함해 다듬는 관심사
- `Interests`는 3개 항목 수는 유지하되, 각 항목 이름을 더 구체적인 실무 언어로 조정한다.

## Copy Principles

- 포트폴리오의 축인 `문제 정의`, `판단 기준`, `검증`, `실제 구현`과 자연스럽게 이어져야 한다.
- 너무 성과 중심으로 무거워지지 않도록 소개 문단에는 수치 대신 작업 방식과 관심 영역을 담는다.
- `Interests`는 키워드 나열이 아니라, 포트폴리오 프로젝트 경험을 연상시킬 정도의 구체도는 확보한다.
- 전체적으로는 여전히 빠르게 읽히는 프로필 톤을 유지한다.

## Expected Result

- GitHub README 첫인상만 읽어도 포트폴리오와 같은 사람처럼 느껴진다.
- 기존보다 내용이 덜 비어 보이면서도, 배지 중심의 가벼운 인상은 그대로 유지된다.
